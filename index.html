git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull origin main: creates new branch, now you'll be 'switched to a new branch 'feature/add-html'
  • If branch is up to date with main, result when type git status will be 'nothing to commit, working tree clean
  • Don't forget to push your branch back into the main branch
  • Use add -A command to commit all files in the working branch.
  • git comment -m "added code to HTML file": after staging files, this commits them in the working branch
  • git pull origin main: makes sure current branch is up to date before merging remote branch in GitHub and local branch
  • push up changes with git push origin feature/add-html
